# 文档边界

## 目的

这个文件只说明“哪类信息应该写在哪里”。根目录负责路由，模块目录负责本地真相。

## 根级文档负责什么

- `README.md`：项目摘要、结构和阅读顺序
- `STATUS.md`：项目级现状、活跃模块、已知偏差
- `AGENTS.md`：根级规则与文档优先级
- `docs/IMPLEMENTATION_PLAN.md`：当前 change 的范围、阶段与验收标准
- `docs/DESIGN.md`、`docs/TOKEN.md`、`docs/UI_MAIN_PAGE.md`：未来 UI 输入，不等于当前交付 gate

## 模块文档负责什么

每个活跃模块都应以本地文档作为第一事实来源：

- `README.md`：模块用途、边界、阅读入口
- `SPEC.md`：当前阶段可确认的本地规范
- `STATUS.md`：已实现、未实现、偏差与不确定项
- `AGENTS.md`：模块级短规则与本地注意事项

## 当前模块划分

- `app/`：唯一应用，负责 UI、host 和应用侧桥接
- `packages/*`：共享模块

## 不该继续堆在根目录的信息

- 应用内部目录拆分
- 共享模块当前公开面
- 单个模块的局部偏差
- 仅对单个模块成立的测试、运行或排障说明

## 当前特殊情况

- 当前版本不包含 `web` 应用文档
- `app/` 当前以边界说明为主，不把未来 UI 结构写成既成事实
